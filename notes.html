📝 Study Notes

Concise, exam-ready notes covering core Digital Electronics topics.

1. Number Systems

Decimal (Base 10): Digits 0–9. Positional notation: 234 = 2×10² + 3×10¹ + 4×10⁰.

Binary (Base 2): Digits 0, 1. Used in all digital systems. Example: 1011₂ = 11₁₀.

Octal (Base 8): Digits 0–7. Useful shorthand for binary (3 bits per octal digit).

Hexadecimal (Base 16): Digits 0–9, A–F. Common in computing (4 bits per hex digit).

Conversion tricks:

  • Binary → Octal: group 3 bits from right
  • Binary → Hex: group 4 bits from right
  • Decimal → Any base: repeated division
  • Any base → Decimal: positional expansion

BCD (Binary Coded Decimal): Each decimal digit encoded as 4-bit binary. Example: 25₁₀ = 0010 0101 BCD.

Excess-3 code: Add 3 to each digit, then encode in binary.

Gray code: Adjacent codes differ by 1 bit only. Used in K-maps and error correction.

2. Logic Gates

Basic gates: AND, OR, NOT

Universal gates: NAND, NOR — any function can be implemented using only these.

Special gates: XOR (inequality), XNOR (equality).

Active levels: Bubbles in symbols indicate inversion (active-LOW).

IC packages:

  • 7400 — Quad 2-input NAND
  • 7402 — Quad 2-input NOR
  • 7404 — Hex NOT (inverter)
  • 7408 — Quad 2-input AND
  • 7432 — Quad 2-input OR
  • 7486 — Quad 2-input XOR

De Morgan's Theorem: (A·B)' = A' + B' and (A+B)' = A'·B'

3. Boolean Algebra

Standard Forms:

  • SOP (Sum of Products): OR of AND terms (minterms)
  • POS (Product of Sums): AND of OR terms (maxterms)
  • Canonical form: Each term contains all variables

Minimization techniques:

  • Boolean algebra theorems
  • Karnaugh Map (K-map)
  • Quine-McCluskey method (tabular)

K-map grouping rules:

  • Group sizes must be powers of 2 (1, 2, 4, 8, 16)
  • Groups can wrap around edges
  • Minimize number of groups and literals

Don't-care conditions: Output values that are never used — can be 0 or 1 to simplify.

4. Adders & Subtractors

Half Adder: Adds 2 bits → Sum (A⊕B), Carry (A·B).

Full Adder: Adds 2 bits + Carry-in → Sum (A⊕B⊕Cin), Cout (AB + BCin + ACin).

Ripple Carry Adder: Chain of full adders. n-bit adder uses n FAs. Delay = n × FA delay.

Carry Look-Ahead Adder: Computes carries in parallel using Generate (G = A·B) and Propagate (P = A⊕B). Faster but more complex.

Subtractor: A − B = A + (2's complement of B).

BCD Adder: Adds BCD digits; if result > 9, add 0110 (6) for correction.

5. Flip-Flops

SR Flip-Flop: Set/Reset. S=R=1 is forbidden (invalid).

JK Flip-Flop: Like SR but J=K=1 toggles. Eliminates invalid state.

D Flip-Flop: Data input captured on clock edge. Qt+1 = D.

T Flip-Flop: Toggle when T=1. Used in counters (each T-FF divides frequency by 2).

Triggering:

  • Level-triggered (Latch): Sensitive to input level during entire clock pulse.
  • Edge-triggered (FF): Sensitive only at clock edge (rising ↑ or falling ↓).
  • Master-Slave: Two latches in series — eliminates race condition.

Characteristic Equations:

  • SR: Qt+1 = S + R'·Qt
  • JK: Qt+1 = J·Q't + K'·Qt
  • D: Qt+1 = D
  • T: Qt+1 = T ⊕ Qt

6. Multiplexer & Demultiplexer

MUX: N inputs → 1 output. Select lines: log₂(N).

Equation: Y = Σ mi·Di (sum of minterm × data).

Application: Implementing any Boolean function of n variables using 2n-1-to-1 MUX.

DEMUX: 1 input → N outputs. Routes input to selected output.

MUX as Universal Logic: 2n-1:1 MUX can implement any n-variable function by feeding constants on one input.

7. Encoders & Decoders

Encoder: 2N inputs → N binary outputs. Active input determines code.

Priority Encoder: Handles multiple active inputs. Highest-priority input wins (often highest index).

Decoder: N inputs → 2N outputs. Only one output is active (corresponds to input code).

BCD-to-7-Segment: 4-bit BCD input → 7 segment control lines (a–g).

Common ICs:

  • 74148 — 8-to-3 priority encoder
  • 74138 — 3-to-8 decoder (also used as 1-to-8 DEMUX)
  • 7447 / 7448 — BCD-to-7-segment decoder

8. Registers & Counters

Register: Group of flip-flops storing N-bit data.

  • Shift Register: Data shifts left or right on each clock (SIPO, PISO, SISO, PIPO).
  • Universal Shift Register: Can shift both directions and load parallel data.

Counter: Sequential circuit that cycles through states.

  • Ripple (Asynchronous): Each FF clocked by previous output. Slow but simple.
  • Synchronous: All FFs clocked together. Faster, common clock.
  • Up Counter: Counts 0, 1, 2, ..., (2n − 1)
  • Down Counter: Counts in reverse
  • Mod-N Counter: Counts 0 to N−1, then resets
  • Ring Counter: One circulating 1; n states for n FFs
  • Johnson Counter: Twisted ring; 2n states for n FFs

Counter Design Steps:

  1. Draw state diagram
  2. Determine number of flip-flops (2n ≥ states)
  3. Choose FF type (JK common)
  4. Derive excitation table
  5. Use K-map to find FF input equations

9. Memory & Programmable Logic

ROM (Read-Only Memory): Permanent storage. Each address maps to fixed data.

RAM (Random Access Memory): Read/write memory. SRAM (fast, flip-flops) vs DRAM (dense, capacitors).

PLA (Programmable Logic Array): Programmable AND and OR arrays.

PAL (Programmable Array Logic): Programmable AND, fixed OR.

PLD (Programmable Logic Device): General term for programmable logic ICs.

FPGA: Field-Programmable Gate Array — large array of configurable logic blocks.

10. A/D & D/A Converters

DAC (Digital to Analog):

  • Weighted Resistor: R, 2R, 4R, 8R...
  • R-2R Ladder: Uses only R and 2R resistors — most common.

ADC (Analog to Digital):

  • Flash ADC: Fastest, uses 2n−1 comparators. Expensive.
  • Successive Approximation: Binary search approach. Good balance of speed and cost.
  • Dual Slope: Very accurate, slow. Used in digital voltmeters.
  • Counter/Ramp: Simple but slow.

Resolution: Step size = VFS / 2n

11. Exam Tips & Common Mistakes

  • Always check for don't-care conditions in K-maps — they simplify logic significantly.
  • Remember De Morgan's: bubble on AND becomes OR with inverted inputs.
  • 2's complement subtraction: A − B = A + (2's comp of B). Discard carry if unsigned.
  • In SR flip-flop, S=R=1 is invalid — not "hold" or "toggle".
  • For 2n-to-1 MUX implementing n-variable function: connect most significant variable to input 0.
  • BCD addition: if sum > 9, add 0110 (6) and check for carry.
  • Excess-128 (8-bit): bias for IEEE 754 exponent.
  • Don't confuse latch (level-triggered) with flip-flop (edge-triggered).
  • Gray code conversion: Gi = Bi+1 ⊕ Bi (binary to gray).
  • Binary to gray: Gi = Bi+1 ⊕ Bi. Gray to binary: Bi = Gi ⊕ Bi+1.
